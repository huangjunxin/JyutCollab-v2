# 視圖、篩選同標示

當詞庫變大後，單靠逐頁翻看很難有效工作。視圖、篩選同標示功能可以幫你把任務轉化成清晰的小集合。

## 何時使用篩選

- 只想看自己有權限的方言點。
- 想處理某個狀態，例如草稿或待審。
- 想找某一類語義場的詞。
- 想檢查缺少讀音、例句或分類的詞條。
- 想找符合某些字形或格式規則的詞。

## 進階篩選

進階篩選適合更精細的資料清理。例如：

- 找出詞頭包含某個字的詞條。
- 找出釋義中可能含有未統一術語的詞條。
- 找出例句格式不一致的資料。
- 用多個條件同時縮窄範圍。

使用進階篩選時，建議先從少量條件開始，確認結果合理後再加入更多條件。

## 正則條件

如果你熟悉正則表達式，可以用它找更複雜的字串模式。

### 常用正則範例

| 目的 | 正則表達式 | 說明 |
|---|---|---|
| 找出可能缺聲調的讀音 | `^[a-z]+$` | 匹配只有字母、沒有數字聲調的粵拼。 |
| 找出含全形括號的詞頭 | `[（）]` | 檢查是否誤用了全形圓括號而非方括號。 |
| 檢查例句缺少句末標點 | `[^。！？~～]$` | 匹配不以常見句末標點結尾的例句。 |
| 找出含普通話拼音的欄位 | `[āáǎàēéěèīíǐìōóǒòūúǔùǖǘǚǜ]` | 檢查是否有人誤填普通話拼音而非粵拼。 |
| 檢查詞頭含數字 | `[0-9]` | 詞頭一般不應包含數字（除非是特殊記號）。 |
| 找出變體格式異常 | `\[.*\[` | 檢查方括號是否有巢狀使用問題。 |

正則條件很強，但也容易過度匹配。批量操作前一定要抽查結果，確認匹配到的確實是需要修正的項目。

## 狀態標示

狀態標示幫助你快速辨認詞條處於哪個階段：

- 草稿：仍在編輯。
- 待審：需要審核員處理。
- 已通過：可作正式材料使用。
- 已拒絕：需要修改或暫不收錄。

## 規則標示

規則標示用來在表格中突出可能需要注意的詞條。以下列出常見的標示類型同處理建議：

| 標示類型 | 觸發條件 | 建議處理方式 |
|---|---|---|
| 缺少讀音 | 詞條已保存但擴展粵拼欄位為空 | 補充方言點讀音；如確實無法確定，先標為草稿。 |
| 缺少例句 | 詞條有義項但沒有例句 | 加入至少一個自然例句。 |
| 缺少分類 | 詞條未選擇主題分類 | 瀏覽或搜索合適的語義分類。 |
| 格式不一致 | 詞頭或讀音格式與系統預期不符 | 檢查變體括號、分隔符是否符合規範。 |
| 疑似重複 | 同一方言點內存在相同或高度相似詞頭 | 打開比對，判斷是否應合併或區分。 |
| 待處理 AI 建議 | 有 AI 建議生成但尚未接受或忽略 | 檢視建議內容，決定採納、修改或略過。 |
| 審核意見未處理 | 已拒絕詞條尚未按拒絕理由修正 | 閱讀拒絕理由，針對性修改後重新提交。 |

看到標示時不要只為了消除提示而修改；應先理解它指出的是資料問題、格式問題，還是審核流程問題。處理完問題後標示會自動消失。

## 建議整理策略

1. 選定一個明確任務，例如「補齊香港待審詞條的例句」。
2. 用方言點、狀態同欄位條件篩選。
3. 按表格逐條修改。
4. 每完成一小批就保存。
5. 最後清除篩選，抽查整體是否有副作用。
