# 詞條欄位

每個欄位都對應一種語言學判斷。良好的詞條不只是「有翻譯」，而是清楚記錄詞形、讀音、用法、義項邊界同方言歸屬。

## 詞頭

詞頭是詞條的主要書寫形式。輸入時請留意：

- 使用香港繁體字形。
- 不要把多個互不等同的詞硬放在同一詞頭。
- 如果是固定多音節表達，盡量保持完整形式。
- 如果有異寫或變體，可以在詞頭欄用 `主詞頭[變體一;變體二]` 格式輸入。

### 詞頭變體格式

詞頭欄會自動解析方括號內的變體：

| 輸入 | 解析結果 |
|---|---|
| `A[B;C]` | 主詞頭是 `A`，變體是 `B`、`C`。 |
| `A[B,C]` | 主詞頭是 `A`，變體是 `B`、`C`。 |
| `A[B；C]` | 全形分號亦會被視為分隔符。 |
| `A［B；C］` | 全形方括號亦可使用。 |

系統會把方括號外的文字保存為主詞頭，方括號內用半形分號、全形分號、半形逗號或全形逗號分隔成多個變體。如果某個變體同主詞頭完全相同，系統會自動忽略該重複項。

括號格式只適合記錄真正可搜尋、可對應的異寫或變體；不要用來寫長備註。

## 方言點

方言點表示該詞條屬於哪一個地理或社羣語言點。它影響：

- 權限檢查。
- 重複詞條判斷。
- 聚合視圖同詞語組比較。
- 審核員如何評估材料來源。

不要把「聽過某地也有人講」直接當成方言點歸屬。最好有語料、受訪者或文獻支持。

## 擴展粵拼（J++）

本系統各地讀音使用 **擴展粵拼（Jyutping++，J++）** 記錄。J++ 以香港語言學學會粵拼為基底，保留一般粵拼的拼式同正字法規則，同時為廣州以外的粵語方言補充更多聲母、韻母、聲調同附加標記。它的定位介乎寬式音位記錄同嚴式語音記錄之間，方便各地材料用同一套 ASCII 方案輸入、檢索同比較。

填寫時應：

- 一般香港 / 廣州粵語可直接使用普通粵拼。
- 其他方言點若有標準粵拼未能表示的音系特徵，按 J++ 規則補充。
- 保持同一方言點內記音體系一致。
- 注意聲母、韻母、聲調、入聲、變調同口語音。
- 不確定時先標記為草稿，不要用普通話拼音或猜測讀音代替。
- 新建詞條填入詞頭後，系統可能會顯示泛粵典 / 字表讀音參考；可採納作為草稿，但仍要按所選方言點核對。

J++ 的核心原則包括：

- **向下兼容**：不修改原有粵拼，擴展部分只作增量補充。
- **通粵泛用**：提供覆蓋粵語方言音系的符號庫，各方言點按需要調用。
- **兼顧歷時同共時**：既可記錄當代方言音系，也可服務跨地域同歷史音系比較。
- **純 ASCII 輸入**：主要使用基礎英文字母、數字同少量 ASCII 標記，方便鍵盤輸入同資料處理。

### 多個讀音格式

擴展粵拼欄會自動把分隔符拆成多個讀音：

| 輸入 | 解析結果 |
|---|---|
| `gong2; gong3` | 兩個讀音：`gong2`、`gong3`。 |
| `gong2, gong3` | 逗號亦可分隔多個讀音。 |
| `gong2；gong3` | 全形分號亦可使用。 |
| `gong2，gong3` | 全形逗號亦可使用。 |

系統保存時會把多個讀音記錄為讀音列表，並以分號形式顯示。只有真正屬於同一詞條的多讀音才應放在同一欄；如果讀音差異其實對應不同詞形、不同方言點或不同詞語，應另行拆分或檢查詞語組。

### 常見 J++ 擴展標記

以下只列入門時最容易遇到的類型；完整規範應以泛粵大典「擴展粵拼」說明為準。

| 類型 | 常見寫法 | 用途 |
|---|---|---|
| 擴展聲母 | `zl`、`cl`、`sl`、`sh`、`sj`、`th` | 記錄不同地點的舌冠、邊擦、齦齶或齒間等對立。 |
| 濁化標記 | `rb` 或 `br`、`tr` | 表示相對應的濁音；部分寫法可前置或後置。 |
| 清響音 | `hm`、`hn`、`hl` | 表示清化的鼻音或邊音等響音。 |
| 內爆音 | `bb`、`dd` | 用於勾漏等片可見的內爆音。 |
| 特殊鼻音 / 塞化鼻音 | `mb`、`nd`、`ngg`、`nj` | 記錄鼻冠塞音、後塞鼻音或齦齶鼻音等。 |
| 喉塞相關 | `q`、韻尾 `-h` | `q` 可記真零聲母與喉塞音對立；喉塞作韻尾時用 `-h`。 |
| 入聲調擴展 | `7`、`8`、`9`、`10` | 用於陰入、陽入及長短分化等情況。 |
| 變調 | `*` | 例如 `caang*2` 可表示變調相關資訊。 |
| 送氣調 | `'` | 例如 `1'` 可標示送氣調。 |
| 明示調值 | `` ` `` | 可在調號後附調值，如 `zung1`55`。 |

J++ 符號庫較寬，並不表示每個方言點都要使用全部符號。常規詞典記錄應在「足以區分該地音系」同「方便檢索、比較」之間取得平衡。

### 泛粵典 / 字表讀音參考

新建詞條時，填入詞頭後系統可能會查詢泛粵典或字表，並在粵拼欄附近顯示讀音參考。這些建議可以幫你快速起草 J++ 讀音，但不等於該方言點的最終判斷。

使用時請留意：

- 讀音參考只是一個候選，仍要核對方言點、語料同項目記音習慣。
- 如果顯示多個地點或多個讀音，應選擇真正屬於目前詞條方言點的形式。
- 若建議來自字表索引或明顯不是讀音資料，系統會盡量忽略；仍有疑問時應手動查核。
- 不應為了採納建議而加入未核實讀音。

## 義項

義項描述詞語的核心意思。好的義項應該：

- 用簡潔香港繁體中文解釋。
- 區分本義、引申義同語用限制。
- 避免只寫普通話對應詞。
- 如果一個詞有明顯不同意思，分成多個義項或分義項。

## 例句

例句用來展示詞語在自然語境中的用法。建議：

- 例句要像真實說話或書寫，不要為了塞詞而造硬句。
- 如有翻譯或說明，應突出語境、搭配同語氣。
- 粗俗、文雅、書面或特定社羣用法要如實標示。

## 語域

語域描述詞語的使用場合，例如：

- 口語。
- 書面。
- 粗俗。
- 文雅。
- 中性。

語域不是道德評價，而是幫助使用者理解「在哪些場合會自然出現」。如果語域欄旁邊出現 **AI 語域** 建議，可以參考其判斷，但仍要按真實語境、例句同方言點用法核對；尤其是粗俗、文雅、書面或特定社羣用法，不應只按 AI 建議決定。

## 主題分類

主題分類把詞條放入語義層級，方便搜尋、統計同語義場研究。選分類時：

- 以詞語主要義項為準。
- 不要只看詞面字義。
- 多義詞可以按主要收錄義項分類。
- AI 分類只作參考，最終由人判斷。

## 狀態

| 狀態 | 意義 | 誰通常處理 |
|---|---|---|
| 草稿 | 尚未完成或尚未提交 | 貢獻者。 |
| 待審 | 已提交，等待審核 | 審核員。 |
| 已通過 | 審核後確認可收錄 | 審核員維護。 |
| 已拒絕 | 審核後需要退回或不收錄 | 貢獻者按理由修正。 |

## 變體同括號形式

詞頭變體主要在詞頭欄用方括號格式記錄，例如 `A[B;C]`。原則是：

- 主詞頭保留最常用或最適合索引的形式。
- 變體應可幫助搜尋，而不是製造混亂。
- 括號只用於真正可替換或補充的成分，不要用來寫長備註。
