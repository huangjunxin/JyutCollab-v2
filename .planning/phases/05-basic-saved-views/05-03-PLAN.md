<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| URL query→client restore | `?view=` is untrusted route input that may be embedded payload or saved view ID. |
| API saved view→filter/rule restoration | Fetched saved state crosses from server persistence into local table overlay state. |
| UI action→clipboard/route | Share action creates URLs and writes to clipboard. |
| saved view selection→table state | Selecting a view changes filters/rules while entries must remain read-only. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-03-01 | Tampering | `applySharedViewQuery()` | mitigate | Discriminate `eyJ` embedded payloads and view IDs; validate embedded payloads with `decodeEntriesSharedView()` and rely on server-validated saved states. |
| T-05-03-02 | Tampering | saved view restoration | mitigate | Restore only through `restoreAdvancedFilterState()` and `replaceRuleOverlayState()`; do not call entry save/delete/bulk APIs. |
| T-05-03-03 | Information Disclosure | share URL generation | mitigate | Share URLs contain only `?view=view_...`, not serialized state, sessions, user IDs beyond public view metadata, or entry data. |
| T-05-03-04 | Denial of Service | route watcher loop | mitigate | Preserve `lastAppliedSharedView` guard for both embedded payload strings and view IDs. |
| T-05-03-05 | Elevation of Privilege | client manage actions | mitigate | Hide edit/delete in UI for non-owners; server-side Plan 05-01 checks remain authoritative. |
</threat_model>
