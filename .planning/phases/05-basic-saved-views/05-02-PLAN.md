<interfaces>
Plan 05-01 creates these API contracts:
- `GET /api/views` returns saved views visible to current authenticated user.
- `POST /api/views` accepts `{ name: string, visibility: 'public' | 'private', state: EntriesSharedViewState }`.
- `PUT /api/views/[id]` accepts owner/admin updates for `name`, `visibility`, and/or `state`.
- `DELETE /api/views/[id]` deletes owner/admin views.
- `GET /api/views/[id]` resolves a shared view ID.

Component event contracts to create:
- Save modal emits `save` with `{ name, visibility, state, id? }` and `cancel`.
- Manage modal emits `apply`, `edit`, `delete`, `copy-as-own`, and `close`.
- Shared banner emits `apply`, `save-as-own`, and `close`.
</interfaces>

| Boundary | Description |
|----------|-------------|
| browser component→API composable | Components emit user-provided names/visibility/state to parent/composable for persistence. |
| saved view state→UI rendering | Saved view names and creator names render in modals/dropdowns/banners. |
| ownership data→client action visibility | Client uses `creatorId` to hide owner-only actions while server remains authoritative. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-02-01 | Tampering | EntriesSaveViewModal | mitigate | Component validates name length and visibility enum before emitting; server validation remains authoritative in Plan 05-01. |
| T-05-02-02 | Elevation of Privilege | EntriesManageViewsModal | mitigate | Hide `編輯`/`刪除` for non-owners per D-20; server owner/admin checks in Plan 05-01 enforce the real boundary. |
| T-05-02-03 | Information Disclosure | UI rendering saved view metadata | accept | Saved view names/creator display names are intended visible metadata for public views; Vue template escaping handles injected markup. |
