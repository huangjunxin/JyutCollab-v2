## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Label | 13px | 400 (regular) | 1.4 |
| Heading | 18px | 600 (semibold) | 1.2 |
| Display | 24px | 600 (semibold) | 1.2 |

**Usage mapping:**
- **Body (14px/regular/1.5):** Modal descriptions, banner descriptions, list item text, empty state body, form field descriptions, error messages
- **Label (13px/regular/1.4):** Form field labels (視圖名稱, 可見性), visibility badges, section labels in dropdown, timestamp text
- **Heading (18px/semibold/1.2):** Modal titles, banner titles, empty state heading, dropdown section headers
- **Display (24px/semibold/1.2):** Reserved for the main entries page "視圖" section header if needed; may also be used as the empty state heading (precedence: heading size at 18px per convention, but empty state heading can use display size per visual hierarchy)

**Edge case:** Empty state heading uses Display (24px) for visual prominence in the empty viewport. The manage modal title uses Heading (18px).

## Color

All colors reference @nuxt/ui CSS variable tokens. The project uses @nuxt/ui's built-in dark/light mode via `@nuxt/ui`'s `ColorModeButton`.

### Light Mode

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--ui-bg` / `white` (#ffffff) | Page background, main content area |
| Secondary (30%) | `--ui-bg-elevated` / `gray-50` (#f9fafb) | Cards, sidebar, nav, modal backgrounds, dropdown menu surface |
| Accent (10%) | `--ui-primary` / `primary-500` | Primary CTA buttons (儲存), active view indicator, focus rings, selected state, link text |
| Destructive | `--ui-error` / `red-500` (#ef4444) | Delete button text, delete confirmation modal CTA, destructive hover states |

### Dark Mode

Colors invert automatically via @nuxt/ui's color mode system. No manual dark mode tokens needed.

### Accent Reserved For

The `--ui-primary` accent color is reserved exclusively for:
- Primary CTA button (`UButton` with `color="primary"`): "儲存" in save modal
- Active/selected saved view indicator in the dropdown
- Focus rings on form inputs and interactive elements
- "套用視圖" action in the shared-view banner
- "複製為我的視圖" link action in the shared-view banner and manage modal

Accent is NOT used for:
- View mode options (flat/aggregated/lexeme) -- these are standard dropdown items
- Visibility badges (use neutral/muted styling)
- Separators and section labels (use muted/neutral)
- "取消" or "關閉" actions (use secondary/ghost variant)

### Destructive Color Reserved For

The `--ui-error` color is reserved exclusively for:
- Delete button text and icon in the manage views modal
- Delete confirmation modal: the "刪除" CTA button
- Destructive hover/active states on delete actions

### Destructive Actions

| Action | Trigger | Confirmation Approach |
|--------|---------|----------------------|
| Delete saved view | "刪除" button in manage modal (owner only) or admin override | Dedicated `UModal` with prompt: 刪除「{viewName}」？此操作無法還原。CTA: 刪除 (destructive variant). Cancel: 取消 (ghost/secondary variant). |

No other destructive actions exist in this phase.

### Error State Display Pattern

All error messages display as inline text or toast notification depending on context:
- **API errors during save/update/delete:** Display as `UAlert` with `color="error"` inside the relevant modal, below the form or at the top of the modal body. Do NOT close the modal on error -- let the user retry.
- **View not found (shared link):** Display as `UAlert` with `color="warning"` in the banner position where the shared-view banner would normally appear. Include a "關閉" action that clears the `?view=` query parameter.
- **Permission errors:** Client-side prevention is primary (hide actions the user can't take). If a server-side permission error occurs despite client guards (e.g., race condition), display as `UAlert` error in the manage modal.

## Interaction States

### Views Dropdown (`EntriesViewsDropdown`)

| State | Behavior |
|-------|----------|
| **Default** | Shows current selection: view mode label (平鋪/按詞形聚合/按詞語聚合) or saved view name. `USelectMenu` with `size="sm"` matching current toolbar style. |
| **Expanded** | Three sections separated by dividers: (1) view modes, (2) saved views grouped by "我的視圖" and "公開視圖" labels, (3) action items "儲存目前視圖..." and "管理視圖...". |
| **My view item** | Shows view name + `UBadge` visibility badge (公開 in green variant, 私人 in muted variant). Own views show a subtle indicator (e.g., person icon). |
| **Public view item (other user)** | Shows view name + `UBadge` (公開) + creator name in smaller muted text. |
| **Empty (no saved views)** | Section labels "我的視圖" and/or "公開視圖" show with "(無)" or are hidden. Action items always visible. |
| **Loading** | While fetching view list, show subtle loading indicator or skeleton in the saved views section. |
| **Error (list fetch fail)** | Show inline error text in dropdown: "載入視圖列表失敗". User can still access view modes and actions. |

### Save Modal (`EntriesSaveViewModal`)

| State | Behavior |
|-------|----------|
| **Default** | `UModal` with title "儲存目前視圖". Name input (pre-filled with empty string, placeholder shown). Visibility radio/toggle defaulting to "私人". |
| **Name validation** | Required field. Min 1 char, max 100 chars. HK Traditional Chinese. Show inline error if empty on submit attempt: "視圖名稱不能為空". |
| **Submitting** | Button shows loading spinner. Form inputs disabled. Modal stays open. |
| **Success** | Modal closes. Toast or brief success indicator. Dropdown refreshes view list. Newly saved view becomes the active selection. |
| **Error** | Inline `UAlert` error at top of modal body with specific message (saveFail). Form remains editable for retry. |
| **Cancel** | Modal closes. No state changed. |

### Manage Modal (`EntriesManageViewsModal`)

| State | Behavior |
|-------|----------|
| **Default** | `UModal` with title "管理視圖". List of user's own views + public views. Each row: view name, visibility badge, creator name (for public), created/updated date, action buttons. |
| **Own view row** | Actions: "編輯" (opens save modal pre-filled for update via PUT), "刪除" (opens delete confirmation modal). |
| **Other's public view row** | Actions: "檢視" (applies the view state), "複製為我的視圖" (opens save modal pre-filled with view state, creates new view). |
| **Empty** | Same empty state as the main empty state (heading + body). |
| **Loading** | Skeleton rows while fetching view list. |
| **Delete confirmed** | View removed from list with transition/animation. If the deleted view was currently active, revert to flat mode and clear `?view=` param. |
| **Error (CRUD fail)** | Inline `UAlert` at modal top with specific error message. Modal stays open. |

### Shared-View Banner (`EntriesSharedViewBanner`)

| State | Behavior |
|-------|----------|
| **Named view (view ID in URL)** | `UAlert` with `color="info"`. Title: 你正在檢視「{viewName}」. Description: 此視圖由 {creatorName} 建立。篩選和規則已套用，不會修改詞條資料。 Actions as text buttons/links within the alert: 套用視圖, 儲存為我的視圖, 關閉. |
| **Anonymous view (base64url in URL, Phase 3 compat)** | `UAlert` with `color="info"`. Title: 你正在檢視分享的視圖. Description: 篩選和規則已套用，不會修改詞條資料。 Actions: 套用視圖, 儲存為我的視圖 (opens save modal pre-filled with decoded state), 關閉. |
| **Dismissed** | Banner removes from DOM. `?view=` query param is cleared from URL. View state persists (user keeps the applied filters/rules until they change them). |
| **View not found** | `UAlert` with `color="warning"`. Title: 視圖不存在或已被刪除. Action: 關閉 (clears `?view=` and dismisses banner). |
| **Re-application guard** | The existing `lastAppliedSharedView` guard prevents banner from re-showing for the same view ID/payload. If the user manually changes filters after applying, the banner does not re-appear. |

### Delete Confirmation Modal

| State | Behavior |
|-------|----------|
| **Default** | Dedicated `UModal` (separate from manage modal). Title/body: 刪除「{viewName}」？此操作無法還原。 Two buttons: 取消 (ghost/secondary variant) and 刪除 (destructive/error variant). |
| **Confirming** | Delete button shows loading spinner. Both buttons disabled. Modal stays open. |
| **Success** | Modal closes. Parent manage modal updates its list. |
| **Error** | Inline error in modal: 刪除視圖失敗，請稍後再試. Allow retry. |

## Layout and Layout States

### Toolbar Integration

The unified views dropdown replaces the current standalone `USelectMenu` at `app/pages/entries/index.vue` lines 176-187. It sits in the same position in the toolbar -- between the existing dialect filter and the advanced filter toggle button. The removed `EntriesShareViewPopover` frees toolbar space; no net increase in toolbar width.

### Modal Z-Index and Focus Management

All modals use @nuxt/ui's built-in focus trapping and z-index layering. If a modal opens another modal (e.g., manage modal -> delete confirmation), the second modal stacks above the first. On close, focus returns to the triggering button in the parent modal.

### Empty State Layout

When the user has no saved views (first visit, all deleted), the manage modal and the dropdown's saved-views section both show the empty state:
- Display heading (24px, semibold): 尚未儲存任何視圖
- Body text (14px, regular): 儲存目前篩選和規則設定為命名視圖，方便日後快速切換或與團隊分享。
- The empty state is center-aligned within the modal content area with 2xl (48px) vertical padding.

### Responsive

All components in this phase are toolbar-embedded or modal-based. No responsive breakpoints differ from the existing entries page behavior. The `USelectMenu` width is fixed at approximately `w-44` (up from current `w-28` to accommodate view names + badges).
