<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client→POST/PUT /api/views | Authenticated user submits untrusted view name, visibility, and serialized rule/filter state. |
| unauthenticated client→GET /api/views/[id] | Public link recipient resolves a view ID into saved filter/rule state. |
| API→MongoDB savedviews | Route handlers persist and read Mongoose documents from MongoDB. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01-01 | Tampering | POST/PUT state payload | mitigate | Validate request bodies with Zod and validate/normalize `state` through the existing `EntriesSharedViewState` v1 encode/decode boundary before persistence. |
| T-05-01-02 | Elevation of Privilege | PUT/DELETE /api/views/[id] | mitigate | Check `view.creatorId === event.context.auth.id` or `event.context.auth.role === 'admin'` before mutation per D-19. |
| T-05-01-03 | Information Disclosure | GET /api/views/[id] | mitigate | Return only `id/name/creatorId/creatorName/visibility/state/createdAt/updatedAt`; never return Mongo `_id`, emails, tokens, sessions, or secrets. |
| T-05-01-04 | Spoofing | Authenticated endpoints | mitigate | Require `event.context.auth` for list/create/update/delete and use server-side session user ID as `creatorId`; ignore any client-supplied creator. |
| T-05-01-05 | Denial of Service | Public GET /api/views/[id] | accept | D-14 intentionally makes this public; view state is non-sensitive and no project rate limiter exists. Keep lookup by indexed `id` only. |
</threat_model>
