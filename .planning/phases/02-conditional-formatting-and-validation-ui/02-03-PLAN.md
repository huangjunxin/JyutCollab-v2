<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Overlay composable -> editable cell rendering | Derived metadata crosses into UI that also handles editing, selection, AI suggestions, and review notes. |
| Entries page rendered rows -> overlay evaluator | Currently visible/loaded entries and display contexts are evaluated by local rules. |
| Rule panel actions -> entries page state | Local rule actions are wired into a page with existing server-backed save/delete/review/bulk functionality. |
| Human browser verification -> execution completion | Visual/read-only behavior must be confirmed in an authenticated local browser session. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03-01 | T | `EntriesEditableCell.vue` overlay class merge | mitigate | Only consume fixed safe class names from Plan 01 metadata; merge classes without replacing existing editing/selection/dirty classes. |
| T-02-03-02 | R | `app/pages/entries/index.vue` rule wiring | mitigate | Wire panel events only to overlay composable actions; static scans forbid rule-related `$fetch`, save, delete, review, and bulk calls. |
| T-02-03-03 | T | Entry data boundary | mitigate | Pass metadata through `cellMeta` prop; do not assign overlay state onto entries or call `col.set` outside existing edit-save path. |
| T-02-03-04 | D | Existing table interactions | mitigate | Do not modify filtered/grouped row collections; derive overlay visible entries from rendered rows and verify flat/aggregated/lexeme behavior manually. |
| T-02-03-05 | I | Rule names in cell tooltips | accept | Rule names are user-authored local UI labels shown back to the same user; no server transmission is added. |
</threat_model>
