<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User form input -> rule draft model | Untrusted names, formulas, regex patterns, flags, and field choices enter local state through component controls. |
| Rule panel UI -> overlay composable actions | Component emits can add, clear, toggle, reorder, or remove local rules. |
| Rule panel -> entries page toolbar | Expanded panel is teleported around toolbar layout and must not disrupt search/filter controls. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-02-01 | T | `EntriesRuleOverlayPanel.vue` field controls | mitigate | Render target/regex fields from the supported `fieldOptions` allowlist; Plan 01 revalidates before adding rules. |
| T-02-02-02 | R | Rule action emits | mitigate | Emits are named local UI events only; static scan forbids `$fetch`, save, delete, review, bulk, `col.set`, and `_isDirty`. |
| T-02-02-03 | I | Formula/regex display | accept | Rule contents are user-entered local UI state; component must not log or transmit them. |
| T-02-02-04 | D | Toolbar layout | mitigate | Use Phase 1 teleport host pattern so expanded panel renders below the toolbar rather than pushing toolbar buttons sideways. |
| T-02-02-05 | S | Icon-only controls | mitigate | Trigger/action buttons use HK Traditional labels or aria-labels; expanded panel wires stable `aria-controls`/`aria-expanded`. |
</threat_model>
