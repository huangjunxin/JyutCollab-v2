## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon/text gaps inside share status badges, inline icon spacing |
| sm | 8px | Toolbar button gaps, popover row gaps, alert action gaps |
| md | 16px | Default panel/card padding, alert padding, copy preview spacing |
| lg | 24px | Stacked sections inside a share popover/modal if used |
| xl | 32px | Empty-state block spacing only |
| 2xl | 48px | Large vertical breathing room for page-level feedback only |
| 3xl | 64px | Not used in Phase 3 implementation |

Exceptions:
- Preserve the existing entries toolbar shell: `mb-4 p-3 bg-white dark:bg-gray-800 rounded-xl shadow-sm border border-gray-200 dark:border-gray-700`.
- Share trigger is icon-only and must match accepted Phase 1/2 toolbar triggers: `size="sm"`, `variant="soft"`, `color="neutral"`, `h-8 w-8 justify-center p-0`.
- Minimum desktop visual target for icon-only share/clear buttons is 32px. On mobile, controls stack or wrap so each actionable text button is at least Nuxt UI `size="sm"` height.
- Expanded feedback inserted below the toolbar uses the same Phase 1/2 panel seam: `border-t border-gray-200 dark:border-gray-700 pt-3 mt-2`.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `white` / `gray-800` dark | Main table surface, toolbar surface, share popover/card background |
| Secondary (30%) | `gray-50`, `gray-100`, `gray-200` / `gray-700`, `gray-800`, `gray-900` dark | Toolbar borders, share metadata blocks, URL preview background, neutral helper text |
| Accent (10%) | Nuxt UI primary green (`primary`, `green-600`, `green-400`, `primary-50`, `primary-900/20`) | Copy share CTA, successful copied/restored badge, focus ring, active shared-view status only |
| Destructive | Nuxt UI error/red (`error`, `red-50`, `red-600`, `red-700`, `red-900/20`) | Invalid/unsupported shared payload feedback only; no destructive action is introduced |

Accent reserved for: `複製視圖連結` primary CTA, successful `視圖連結已複製` feedback, `已套用分享視圖` status badge, active focus rings on share controls, and existing page title icon treatment. Do not use accent for all toolbar icons, neutral helper copy, raw URL preview text, invalid payload warnings, or validation warnings from Phase 2.

**Semantic colors:**
- Invalid, unsupported, too old, or failed-to-decode shared view payloads use red/error with `role="alert"` and must not clear the table.
- Partial restore or ignored unknown fields use warning/amber only if the supported parts were applied; otherwise use red/error.
- Read-only/safety helper copy remains neutral/gray.
- Validation rule warnings from Phase 2 remain amber and visually distinct from Phase 3 payload errors.

## Layout Contract

### Placement

Add shareable-view UI to the existing search/filter card in `app/pages/entries/index.vue`.

Required structure:
1. Keep existing plain search, server filters, view mode selector, advanced-filter trigger, rule trigger, and column-width controls in their current hierarchy.
2. Place the share trigger beside the accepted Phase 1 `進階篩選` and Phase 2 `規則` icon-only triggers, before the column-width utility buttons where practical.
3. Use a compact icon-only button with Heroicon `i-heroicons-share` or `i-heroicons-link`; tooltip/aria-label text is `分享目前視圖`.
4. If there is a restored shared view, show a compact `UBadge color="primary" variant="soft"` next to the share trigger with copy `已套用分享視圖`.
5. If the query payload is invalid, show an inline `UAlert color="error" variant="subtle"` below the toolbar row using the Phase 1/2 `border-t ... pt-3 mt-2` expansion area. Do not use browser `alert()`.
6. Do not create a new sidebar, full-screen wizard, or saved-views management screen in Phase 3.

### Share Popover/Card

When the user opens share UI:
- Preferred presentation is `UPopover` anchored to the share trigger. `UModal` is acceptable only if clipboard fallback text is too cramped on narrow screens.
- Width: `w-80` on desktop popover, full available width on mobile; no horizontal overflow.
- Contents in order:
  1. Heading `分享目前視圖`.
  2. Body helper explaining read-only restore.
  3. Version row `分享格式版本：v{version}`.
  4. Optional compact summary: formula active/inactive, regex active/inactive, conditional formatting rule count, validation rule count.
  5. Primary CTA `複製視圖連結`.
  6. Clipboard fallback URL preview only after copy failure or if Clipboard API is unavailable.

### Density

- Trigger: icon-only `UButton size="sm" color="neutral" variant="soft"` with `h-8 w-8 justify-center p-0`.
- Primary copy button: `UButton color="primary" variant="solid" size="sm"`.
- Secondary clear-query action: `UButton color="neutral" variant="soft" size="sm"`.
- Metadata blocks: rounded `bg-gray-50 dark:bg-gray-900/50 border border-gray-200 dark:border-gray-700 p-3`.

## Control Contract

| Control | Component | Required Behavior |
|---------|-----------|-------------------|
| Share trigger | `UButton` inside `UTooltip` | Opens the share popover; never mutates entry data. |
| Copy link CTA | `UButton color="primary" variant="solid" size="sm"` | Serializes current supported view state, writes URL to clipboard, shows success/failure feedback. |
| URL preview fallback | readonly `UInput` or bordered `code` block | Displays the generated URL only when manual copy is needed; text is selectable. |
| Restored status badge | `UBadge color="primary" variant="soft"` | Indicates a valid shared view has been applied. |
| Invalid payload alert | `UAlert color="error" variant="subtle"` | Explains the problem and next step; table remains usable. |
| Clear share parameter | `UButton color="neutral" variant="soft" size="sm"` | Removes only the share-view route/query parameter from the URL; does not clear in-memory filters/rules unless user separately clears them through existing controls. |

### Serialized State Scope

The share link must include only supported read-only table view configuration:
- Phase 1 advanced formula text and applied formula state.
- Phase 1 global regex enabled state, pattern, and supported flags.
- Phase 1 column regex field, pattern, and supported flags.
- Phase 2 conditional formatting rules: name, enabled state, order, target fields, condition kind, formula or regex condition, style preset, color hex.
- Phase 2 validation rules: name, enabled state, order, target fields, condition kind, formula or regex condition.
- Explicit serialization version, starting at `1` for Phase 3.

The share link must not include:
- Entry data, unsaved cell values, dirty drafts, selected row IDs, user IDs, auth/session data, tokens, API responses, Cloudinary credentials, AI suggestions, localStorage draft data, or any destructive/bulk action.

## State Contract

### No Share State

When no supported formula, regex, conditional formatting, or validation configuration exists:
- Share trigger remains visible and enabled.
- Opening the share UI shows the empty copy: `未有可分享設定` and `目前沒有公式篩選、正則設定、條件格式或驗證規則。請先設定視圖，再複製分享連結。`
- `複製視圖連結` is disabled until at least one supported view setting exists.
- Existing table behavior is unchanged.

### Copy Success State

After successful copy:
- Keep the popover open or show a transient success text inside it; do not navigate.
- Show `視圖連結已複製。` as text feedback with primary/green styling.
- Do not show a global modal or browser alert.
- Focus remains on the copy button or returns to the share trigger when the popover closes.

### Copy Failure State

If clipboard write fails:
- Show `無法複製連結。請手動複製下方網址。`
- Reveal a selectable readonly URL preview.
- Keep normal table use available.

### Valid Shared Link Restore State

When opening `/entries` with a valid supported shared-view payload:
- Validate and decode before applying anything.
- Apply supported state atomically: formula/regex settings and Phase 2 rules become visible in their existing panels when opened.
- Show `已套用分享視圖` badge in the toolbar.
- Show a non-blocking summary text if space allows: `已還原 {filterCount} 項篩選和 {ruleCount} 項規則。`
- Do not mark any entry dirty.
- Do not trigger save/delete/review APIs.
- Do not overwrite local draft recovery data.
- Do not change current auth state, dialect permissions, or server-backed filters unless the serialized schema explicitly supports those filters in a future version.

### Invalid Shared Link State

When payload is missing, malformed, too large, unsupported, too old, or contains unsupported fields/rules:
- Fail safely before applying any shared state.
- Keep table usable with existing route/search/default state.
- Display the error state: `分享視圖無法套用：{reason}。已保留目前表格，請清除網址中的分享參數或重新複製視圖。`
- Include `清除分享參數` action.
- Do not clear current local filters/rules if they were already present before route change.
- Do not refetch repeatedly because of the invalid payload.

### Partial Compatibility State

For version `1`, unsupported fields or rule kinds are blocking errors, not partial apply. Future versions may migrate, but Phase 3 must prefer rejection over silently applying an ambiguous shared view.

## Interaction Contract

### Keyboard

- Existing entries table keyboard navigation remains unchanged when focus is inside the table.
- Share trigger participates in normal tab order after rule/filter controls and before column-width utility buttons where placed.
- `Enter` or `Space` on the share trigger opens the popover.
- `Escape` inside the share popover closes the popover and returns focus to the share trigger.
- `Tab` remains trapped only if a modal fallback is used; popover version follows normal DOM tab order.
- URL preview fallback must be keyboard selectable/copyable.
- Table `Arrow`, `Enter`, `Tab`, and `Escape` handlers must not intercept keystrokes while focus is inside share controls.

### Route and Restore Timing

- Decode and validate the shared payload once during initial route handling before applying to filter/rule state.
- If `route.query` changes to a different shared payload while already on `/entries`, validate the new payload and apply atomically only if valid.
- Applying a shared view must not produce an infinite route watcher loop.
- Copying a link must not automatically mutate the current route unless the user opens the copied URL.

### Feedback Timing

- Copy success feedback appears within 300ms of Clipboard API success.
- Copy failure feedback appears immediately after the rejected clipboard promise.
- Invalid shared payload feedback appears before or alongside the first table render after `/entries` loads; loading the table must not be blocked by a bad payload.
- Restored status badge remains visible until the share query parameter is cleared or the page is left.

## Accessibility Contract

- Share trigger has `aria-label="分享目前視圖"` and tooltip text matching the accessible name.
- Popover heading is programmatically associated with content if Nuxt UI supports it; otherwise the heading is visible and first in content.
- Invalid shared-payload feedback uses `role="alert"` or `aria-live="assertive"`.
- Copy success/failure feedback uses `aria-live="polite"`.
- Badge/status text communicates restored state textually; do not rely on green color alone.
- Clear share parameter action includes helper copy explaining it only removes URL parameters.
- URL preview uses a readable monospace or default 14px text, wraps or scrolls within its own container, and remains selectable.
- Do not remove focus outlines or primary focus ring styles.

## Responsive Behavior

### Desktop (`lg` and above)

- Share trigger sits in the compact toolbar icon group near `進階篩選` and `規則`.
- Popover opens below/right of trigger with width around 320px.
- Restored badge may appear inline beside the trigger; truncate only if necessary, but keep full text available by tooltip/title.
- Invalid payload alert spans full toolbar width below the first row.

### Tablet (`sm` to `lg`)

- Toolbar controls may wrap; share trigger stays adjacent to advanced/rule triggers if wrapping permits.
- Popover width uses available viewport minus 32px side margin.
- URL preview wraps or horizontally scrolls inside its preview block only.

### Mobile (`< sm`)

- Preserve existing mobile warning behavior.
- Share UI remains available for browsing and copying; it must not imply mobile table editing support.
- Stack popover/modal content vertically with full-width copy and clear buttons.
- Restored/invalid feedback appears above the table and below the filter toolbar; no horizontal-only controls.

## Implementation Constraints

### Must Preserve

- Existing plain search semantics and `searchQuery` behavior.
- Existing server-backed filters for user, dialect, theme, status, sorting, pagination, and view mode.
- Existing Phase 1 advanced filter panel and behavior.
- Existing Phase 2 rule overlay panel and accepted behaviors: create, toggle, remove, reorder, recolor, validation warning styling.
- Existing grouping modes: flat, aggregated, lexeme.
- Existing inline editing, save/submit/delete, local draft recovery, AI suggestions, keyboard navigation, and column resizing.
- Existing localStorage draft watch over source `entries`, `aggregatedGroups`, and `lexemeGroups`.

### Must Not Build in Phase 3

- Server-side saved views.
- Account/team view management.
- Permissioned view sharing.
- Backend persistence of view state.
- Bulk mutation, bulk preview, or formula-driven entry updates.
- New condition formatting styles beyond Phase 2 presets.
- Spreadsheet grid replacement.
- Third-party UI registry blocks.

### Safety Constraints

- Serialized payload must be parsed as data only. Do not use `eval`, `new Function`, dynamic imports, executable user-provided code, or arbitrary JavaScript execution.
- Validate decoded shared payload with an explicit schema before applying.
- Enforce a maximum encoded payload length before decode; recommended cap is 6,000 URL characters for the share parameter.
- Enforce Phase 1 regex safety limits and supported flags when restoring regex settings.
- Reuse Phase 1 formula/regex validation before applying restored filters or rules.
- Reject unsupported version/fields/rules with clear feedback.
- Never serialize secrets, user/session data, entry data, or unsaved draft content.
