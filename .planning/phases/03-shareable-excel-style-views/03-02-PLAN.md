<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Page-generated URL -> clipboard UI | A generated URL containing user-authored rule/filter text is displayed and copied by browser UI. |
| Invalid shared-view state -> alert component | Decode/validation failure reason becomes user-facing text. |
| Share UI controls -> page state | Component emits actions that the page will handle; the component itself must not mutate trusted app state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-02-01 | I | `EntriesShareViewPopover.vue` URL preview | mitigate | Display only the generated share URL provided by page state; no entry data/session/user/token fields are accepted as props. Keep URL preview readonly/selectable and visible only for fallback states. |
| T-03-02-02 | T | Component action emits | mitigate | Component is presentational and emits `copy` / `clear-share-query` only; tests block `$fetch`, localStorage, route navigation, save/delete/review/bulk, `col.set`, and `_isDirty`. |
| T-03-02-03 | R | Copy feedback | mitigate | Show explicit `視圖連結已複製。` or `無法複製連結。請手動複製下方網址。` with `aria-live="polite"`; avoid silent clipboard failure. |
| T-03-02-04 | D | Oversized or invalid payload UX | transfer | Plan 03-01 utility enforces length/schema; this component only displays supplied `sharedViewError` using `UAlert role="alert"` and must not attempt decode itself. |
| T-03-02-05 | S | Restored status spoofing | mitigate | Restored badge is controlled by the page after successful validation; component displays text `已套用分享視圖` and does not infer trust from URL content. |
</threat_model>
