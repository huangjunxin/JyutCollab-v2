<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser URL -> entries page route watcher | Untrusted `view` query value enters page state on initial load and client-side navigation. |
| Shared-view utility -> composable restore calls | Validated payload crosses into local formula/regex/rule state owners. |
| Clipboard API -> browser permissions | User-triggered copy can fail because browser denies clipboard access or secure context is unavailable. |
| Clear query action -> Nuxt router | User action changes URL query while preserving in-memory table state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-03-01 | T | `applySharedViewQuery` | mitigate | Decode/validate through `decodeEntriesSharedView` before applying; unsupported versions/fields/rules/conditions and invalid formula/regex are blocking errors with no partial restore. |
| T-03-03-02 | D | route watcher | mitigate | Use `lastAppliedSharedView` and keep `view` restore separate from `applyUrlParams()` return value so shared-view changes do not cause infinite loops or unnecessary `fetchEntries()` calls. |
| T-03-03-03 | I | `copySharedViewLink` | mitigate | Build URL from exported read-only state only; do not include entry data, unsaved drafts, selected row IDs, user/session data, tokens, API responses, AI suggestions, or localStorage data. |
| T-03-03-04 | R | invalid/copy feedback | mitigate | Surface invalid restore and copy failure with inline HK Traditional feedback and status state; no silent failures or browser `alert()`. |
| T-03-03-05 | E | restored formulas/regex | transfer | Plan 03-01 utility mitigates executable payload and regex safety; page must only call composable restore after utility success and must not bypass utility validation. |
| T-03-03-06 | S | shared link access control | accept | Shared views only alter local read-only view configuration over entries the current authenticated user can already access; page must not change auth state, dialect permissions, or server-backed filters from v1 payload. |
</threat_model>
