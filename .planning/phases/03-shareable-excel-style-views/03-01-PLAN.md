<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| URL query `view` -> client decode utility | Untrusted user-controlled string becomes candidate shared-view data. |
| Decoded payload -> formula/regex validators | Untrusted formula and regex strings attempt to influence local filtering/rule evaluation. |
| Composable restore APIs -> entries table state | Validated read-only configuration updates local UI state that affects visible rows and overlays. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01-01 | T | `app/utils/entriesSharedView.ts` | mitigate | Validate decoded data with strict Zod schemas at every object boundary; reject unknown keys, unsupported fields, unsupported rule kinds, unsupported condition kinds, unsupported versions, and too-old versions before any composable restore. |
| T-03-01-02 | E | `app/utils/entriesSharedView.ts` formula handling | mitigate | Reuse `parseAdvancedFormula`; never use `eval`, `new Function`, dynamic imports, or executable user-provided code. |
| T-03-01-03 | D | `app/utils/entriesSharedView.ts` regex/length handling | mitigate | Enforce `ENTRIES_SHARED_VIEW_MAX_ENCODED_LENGTH = 6000` before decode and reuse `compileAdvancedRegex` for pattern length, supported flags, invalid pattern, and unsafe pattern rejection. |
| T-03-01-04 | I | `encodeEntriesSharedView` / `export*State` | mitigate | Serialize only read-only filter and rule configuration; tests assert no entry data, unsaved drafts, selected row IDs, user IDs, auth/session data, tokens, API responses, Cloudinary credentials, AI suggestions, or localStorage draft data appear in payloads. |
| T-03-01-05 | R | Restore result reporting | mitigate | Return explicit `{ ok, reason, code }` results with exact HK Traditional messages so invalid payloads are observable and auditable in UI without crashing. |
</threat_model>
