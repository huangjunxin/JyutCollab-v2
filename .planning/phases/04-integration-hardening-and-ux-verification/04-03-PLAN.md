<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser tester -> live `/entries` data | Manual actions can edit real or local development entries if credentials point at a real environment. |
| Shared links -> URL query | Manual payload tests intentionally load trusted and malformed `view` query strings. |
| Clipboard/API/AI/environment -> verification status | External services may be unavailable or credential-limited. |
| Documentation updates -> project status | Marking requirements complete affects future planning and must reflect evidence. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-03-01 | Tampering | manual editing checks | mitigate | Use development/local data where possible; perform only baseline edit/save flows required by SAFE-02 and record environment. No destructive Phase 04 actions are introduced. |
| T-04-03-02 | Information Disclosure | shared-view manual links | mitigate | Use test content only and verify URLs do not contain entry data, drafts, users, tokens, API responses, or secrets. |
| T-04-03-03 | Repudiation | verification docs | mitigate | Record exact row statuses, limitations, failed observations, commands run, and browser environment. |
| T-04-03-04 | Denial of Service | malformed payload/performance tests | mitigate | Use bounded payload examples from existing utility limits; do not perform stress or DoS testing beyond typical loaded data responsiveness. |
| T-04-03-05 | Spoofing | auth/session-dependent tests | mitigate | Record the authenticated role/session used; mark rows `受環境限制` if credentials are missing. |
</threat_model>
