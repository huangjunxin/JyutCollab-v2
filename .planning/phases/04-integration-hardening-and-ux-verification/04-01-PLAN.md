<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| URL query `view` -> `decodeEntriesSharedView()` | User-controlled input becomes candidate view configuration. |
| Decoded view configuration -> composable restore APIs | Validated filter/rule state changes local UI configuration. |
| Rule/filter evaluation -> loaded entries | Inspection code reads current entries and returns filtered rows/overlay metadata. |
| Page wiring -> mutation APIs | Entries page contains edit/save/delete/review/AI/upload capabilities that Excel tools must not call. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01-01 | Tampering | `entriesSharedView.ts` | mitigate | Keep strict schemas, semantic formula/regex validation, version gates, max-length check, and strict `colorHex` validation before restore. |
| T-04-01-02 | Information Disclosure | shared-view encode/export APIs | mitigate | Tests assert exported payloads omit entries, drafts, selected rows, users, sessions, tokens, API responses, Cloudinary data, and AI suggestions. |
| T-04-01-03 | Elevation of Privilege | entries page wiring | mitigate | Source tests assert Excel-tool wiring does not call save/submit/delete/review/history/AI/upload/admin/bulk mutation APIs. |
| T-04-01-04 | Denial of Service | malformed URL payloads | mitigate | Decode length limit and fail-closed parse path prevent large or malformed payloads from reaching rule/filter evaluation. |
| T-04-01-05 | Repudiation | invalid restore feedback | mitigate | Invalid restore returns explicit code/reason and page-level `role="alert"` copy visible outside hidden popovers. |
</threat_model>
